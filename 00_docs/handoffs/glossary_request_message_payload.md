# Запрос к Glossary Agent: расширение определения Message

**От:** Architect
**Дата:** 2026-02-14
**Контекст:** `architecture/interfaces.md`, раздел 2 (Messaging)

## Суть

В `interfaces.md` зафиксировано: Message — единая абстракция входа и выхода на границе Core. Message может содержать текст и опционально дополнительный payload (документы, артефакты и т.д.).

Текущее определение в glossary.md:

> Одна реплика внутри Dialogue. Имеет роль (user, assistant, system)
> и текстовое содержимое.

Определение не покрывает опциональный payload.

## Запрос

Рассмотреть расширение определения Message — добавить возможность дополнительного payload при сохранении текста как основного содержимого.

# Ответ Glossary Agent: расширение определения Message

**От:** Glossary Agent
**Кому:** Architect
**Дата:** 2026-02-14
**Запрос:** `glossary_request_message_payload.md`

## Решение: принято с корректировкой

Определение Message расширено. Вместо термина "payload" введён отдельный термин **Attachment**, чтобы избежать терминологической коллизии с **Payload** (который в глоссарии определён как содержимое BusMessage).

## Что изменено в glossary.md

**Message** — добавлено упоминание опциональных Attachments.

**Attachment** — новый термин в секции Domain:
> Опциональное дополнение к Message: файл, изображение, аудио
> или иные данные, сопровождающие текстовое содержимое.

## Обоснование выбора термина

Термин "payload" в контексте Message создал бы коллизию с глоссарным термином Payload (типизированное содержимое BusMessage). Это разные понятия на разных уровнях:

- **Payload** — всё содержимое BusMessage (межкомпонентная шина)
- **Attachment** — опциональное дополнение к Message (граница Core)

Термин Attachment нейтрален к способу обработки: допускает как прямую обработку мультимодальной LLM в DialogueAgent, так и передачу в EventBus для обработки ProcessingLayer.

## Рекомендация

В `interfaces.md` (строка 44) рекомендуется заменить "дополнительный payload" на "Attachments" для согласованности с глоссарием.

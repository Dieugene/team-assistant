# Материалы к задаче 003: замечания по целевому интерфейсу VS UI

## Источник

Замечания зафиксированы от пользователя при приемке сквозного прототипа после завершения 001.

## Целевые требования (для включения в task_brief_01 задачи 003)

## 1) Трехпанельный layout

- Левая панель: список/контекст диалогов.
- Центральная область: таймлайн событий.
- Правая панель: свойства/детали выбранного объекта (event/dialogue/message/agent action).

## 2) Таймлайн должен выглядеть как таймлайн

- Недостаточно простой последовательности карточек.
- Нужна визуальная структура времени (ось/иерархия/группировка), читаемая как timeline.
- Сохранить вертикальную читабельность при росте количества событий.

## 3) Рассмотреть готовые библиотеки таймлайнов

- Обязательно сделать сравнительный анализ минимум 2-3 библиотек.
- Отдельно проверить ограничение на вертикальный layout (исторически было сомнение, что это узкое место).
- Если библиотека не подходит, зафиксировать это явно и обосновать выбор кастомной верстки.

## 4) Представление payload/event details

- Нельзя показывать только обрезанный сериализованный JSON как основной контент карточки.
- Полезная часть сообщения не должна теряться из-за обрезки.
- Должны быть:
  - компактный preview в timeline;
  - раскрытие/панель с полными деталями (структурировано по полям, а не "сырая строка JSON").

## Предлагаемые Acceptance Criteria для будущей 003

- [ ] AC-UI-1: Реализован трехпанельный layout (Dialogues / Timeline / Properties).
- [ ] AC-UI-2: Timeline имеет явную визуальную модель времени, а не только стек карточек.
- [ ] AC-UI-3: Сделан и задокументирован анализ библиотек таймлайнов; выбор обоснован.
- [ ] AC-UI-4: Для событий есть preview + полный detail view без потери содержания message/payload.
- [ ] AC-UI-5: Выбор события в центре обновляет правую панель свойств.
- [ ] AC-UI-6: Выбор диалога слева фильтрует/фокусирует события таймлайна.

## Примечание по плану

Эти требования относятся к Iteration 3 (`Полноценная наблюдаемость`) и не блокируют завершение 001 или старт 002.
